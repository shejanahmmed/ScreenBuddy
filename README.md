## What is ScreenBuddy?

ScreenBuddy is a **high-performance, peer-to-peer screen mirroring system** that streams your Windows desktop to an Android device over your local network вҖ” with hardware-accelerated H.264 encoding, sub-second latency, full touch input control, and zero reliance on any external server, cloud service, or third-party relay.

It is not a remote desktop tool wrapped in a convenience layer. It is built from the ground up using low-level OS primitives: **DXGI Desktop Duplication** for frame capture, **Media Foundation** for GPU-accelerated encoding, and **Android MediaCodec** for hardware decoding вҖ” delivering a streaming pipeline that operates as close to the metal as possible.

## Key Features

| Feature | Implementation |
|---|---|
| **Screen Capture** | DXGI Desktop Duplication API вҖ” GPU-side frame access, no GDI overhead |
| **Video Encoding** | Windows Media Foundation H.264 encoder вҖ” hardware-accelerated on supported GPUs |
| **Video Decoding** | Android MediaCodec H.264 decoder вҖ” renders directly to `SurfaceView`, zero copy |
| **Auto-Discovery** | UDP broadcast on port `7891` вҖ” no manual IP entry required |
| **Security** | 6-digit PIN handshake вҖ” generated fresh per session, never stored |
| **Touch Control** | Win32 `SendInput` вҖ” bi-directional touch event forwarding from Android to PC |
| **Transport** | Raw TCP вҖ” minimal framing overhead, operates entirely on LAN |

## Requirements

### Windows (Server)
- Windows 10 or later (DXGI Desktop Duplication requires Windows 8+)
- [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8)
- x64 architecture
- GPU with Direct3D 11 and H.264 hardware encoding support (Intel Quick Sync, NVIDIA NVENC, AMD VCE)

### Android (Client)
- Android 7.0 Nougat (API 24) or higher
- Device on the same local Wi-Fi network as the Windows host

## Getting Started

### 1 вҖ” Clone the repository

```bash
git clone https://github.com/shejanahmmed/ScreenBuddy.git
cd ScreenBuddy
```

### 2 вҖ” Run the Windows Server

```bash
cd windows-app/ScreenBuddyCapture
dotnet run
```

The server will:
1. Begin broadcasting its presence on your LAN via UDP
2. Display a freshly generated **6-digit PIN** in the terminal
3. Wait for an authenticated Android client connection

### 3 вҖ” Build & Install the Android App

```bash
# From the project root
./gradlew assembleDebug
adb install app/build/outputs/apk/debug/app-debug.apk
```

Or open the project in **Android Studio** and run it directly on your device.

### 4 вҖ” Connect

1. Launch ScreenBuddy on your Android device
2. The app will auto-discover the PC on your network
3. Enter the 6-digit PIN shown in the server terminal
4. Your PC screen streams directly to the device вҖ” touch the display to control your PC

---

## Security Model

ScreenBuddy uses a **session-scoped PIN** as its authentication mechanism. The PIN is:

- Generated cryptographically fresh on every server launch
- Never persisted to disk or transmitted in plaintext before authentication
- Required to complete the TCP handshake вҖ” unauthenticated clients are immediately disconnected

This model is intentionally minimal. ScreenBuddy is designed for **private LAN use** вҖ” it is not intended to be exposed to the public internet.
